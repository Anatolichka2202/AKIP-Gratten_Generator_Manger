# Оценка навыков разработчика Tolik — AKIP/Gratten Manager

> Дата: 2026-06-24 | Проект: Qt 6.8, C++17, SCPI, USB/LAN

---

## 1. Сильные стороны

**Предметная область и железо**
Самостоятельная интеграция с CH375 USB-чипом через Windows DLL SDK — нетривиальная задача. Грамотный обход конфликтов `min/max` из `windows.h` через `NOMINMAX` + `#undef`. Рабочий LAN/TCP транспорт с буферизацией, таймаутами и retry-логикой.

**Qt-механики на практическом уровне**
Правильный синтаксис сигналов/слотов через указатели на методы (`QOverload<...>::of`), RAII через QObject-парентинг, корректная инициализация через initializer list. Понимание жизненного цикла объектов Qt.

**Задумка интерфейса (IAkipController)**
Создан чистый абстрактный интерфейс с разделением по ответственности: открытие, высокоуровневые команды, FM/PM/Sweep/PULM модуляции, query-методы, низкоуровневый доступ. Это осознанный дизайн, а не уровень начинающего.

**Попытка тестирования**
Написаны тесты с `TestableGrattenGa1483Controller`, паттерн Subclass+Override для инъекции зависимостей. Понимание unit testing в Qt присутствует.

**SCPI-протокол**
Правильное форматирование команд (`\r\n`), понимание особенностей: запрос перед записью, верификация через query-after-set, retry при пустом ответе.

## 2. Зоны роста

**BUG-001..007: хардкод канала в командах**
Канал B дублировал A через хардкод в 7 местах — недостаточный attention-to-detail и отсутствие параметризованных тестов.

**QSettings непоследовательно**
Знание QSettings есть, но применяется непоследовательно. Любая конфигурируемая константа должна идти в QSettings автоматически.

**i18n отсутствовала**
`tr()` не использовался. Это не сложно технически, но требует дисциплины с первого дня написания UI-строк.

**Монолитный MainWindow**
Один огромный файл вместо composable виджетов (ChannelWidget / GrattenControlWidget) — нарушение SRP.

**Uninitialized return value**
Функция `set_power()` возвращала неинициализированный `int` — UB в C++17. Ловится флагом `-Wuninitialized`. Значит, или флаги не включены, или предупреждения игнорируются.

**Тесты: только happy path**
Нет тестов на: потерю соединения в середине транзакции, неверный формат ответа, одновременную команду и disconnect.

**Деплой не настроен**
Изначально не было windeployqt / static build понимания. Нормально для embedded-разработчика без релизного опыта, но для Senior — обязательно.

## 5. Вопросы для устной проверки

**Q1.** В `LanInterface` буфер читается через `+=` в `onSocketReadyRead()`. Что произойдёт, если прибор пришлёт два ответа подряд быстрее обработки событий Qt?

*Ответ:* Оба попадут в `m_readBuffer` слитно. `indexOf('\n')` обрежет первый правильно — это назначение буферизации. Проблема: если первый ответ без `\n` — оба смержатся. Решение: frame-разделитель или длина пакета.

**Q2.** В `IAkipController` деструктор `virtual ~IAkipController() {}` — почему не `= 0`? Что будет без `virtual` при удалении через указатель базового класса?

*Ответ:* Чисто-виртуальный деструктор (`= 0`) должен иметь тело — иначе линковочная ошибка. Без `virtual` при `delete basePtr` вызовется только деструктор базового — утечка ресурсов. Правильно: `virtual ~IAkipController() = default;`

**Q3.** BUG-001..007 — канал B дублировал A. Напишите одну строку теста, которая бы это поймала.

*Ответ:*
```cpp
QVERIFY(m_mock->lastCommand().contains("CH2"));
```
При вызове `setFrequency(2, 1e6)` команда должна содержать идентификатор канала 2. Параметризованный тест по channel=1 и channel=2 поймал бы это с первого запуска.

**Q4.** В `AkipFacade` есть `QMap<int, double> m_freqCache`. В каких ситуациях кэш даст неверный результат?

*Ответ:* Если параметр изменён не через этот экземпляр AkipFacade (другой компьютер, ручное изменение с прибора). Решение: инвалидировать кэш по таймауту или делать query для критичных операций.

**Q5.** `QThread::msleep(100)` в цикле retry — вызывается из UI-потока. Что происходит? Как правильно решить без блокировки?

*Ответ:* UI зависает на время retry (до 300ms при 3 попытках). Решения: 1) QThread/QThreadPool для IO; 2) QTimer::singleShot + конечный автомат; 3) Qt Concurrent. Минимальный fix сейчас: `QApplication::processEvents()` в цикле ожидания.
