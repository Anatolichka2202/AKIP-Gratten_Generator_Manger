# Урок 2: Протокол SCPI — как общаться с приборами

## Что такое SCPI?

**SCPI** (Standard Commands for Programmable Instruments) — это стандарт языка команд для управления измерительными приборами. Принят в 1990 году, основан на IEEE 488.2.

Суть простая: ты отправляешь строку текста, прибор отвечает строкой текста. Никакого бинарного протокола.

---

## Структура команды SCPI

```
:FREQuency:CW 1000000000Hz
  ↑           ↑
  иерархия    значение
```

Команды образуют **дерево** (как папки на диске):
```
:SOURce
  :FREQuency
    :CW           ← несущая частота
  :POWer
    :LEVEL        ← мощность
  :AM
    :STATe        ← AM вкл/выкл
    :DEPTh        ← глубина AM
    :INTernal
      :FREQuency  ← частота модуляции
```

**Правило сокращений**: SCPI позволяет писать как полное имя, так и заглавные буквы:
- `:FREQuency:CW` = `:FREQ:CW` = `:frequency:cw` (регистр не важен)

---

## Set vs Query

В SCPI каждая команда обычно имеет две формы:

```
:POWer:LEVEL -10DBM       ← Set (установить значение)
:POWer:LEVEL?             ← Query (запросить текущее значение)
```

Знак `?` превращает команду в запрос. Прибор отвечает числом или строкой.

---

## Общие IEEE 488.2 команды

Они начинаются с `*` и работают на всех SCPI-приборах:

| Команда | Что делает |
|---|---|
| `*IDN?` | Вернуть идентификационную строку прибора |
| `*RST` | Сброс в заводские настройки |
| `*CLS` | Очистить регистр ошибок |
| `*OPC?` | Ответить "1" когда все команды выполнены |
| `*TST?` | Запустить самотест, вернуть 0 (ок) или код ошибки |

```
*IDN? → "Atten,GA1483,SN12345678,V1.00.00"
```

---

## Транспорты

SCPI-команды можно передавать через разные интерфейсы:

| Интерфейс | Пример | Используется |
|---|---|---|
| GPIB (IEEE-488) | Классика | Старые приборы |
| RS-232 | COM-порт | Устаревает |
| USB (USBTMC) | CDC или HID | АКИП-3417 (через CH375) |
| LAN (VXI-11) | TCP порт 5025 | Gratten GA1483 |
| USB (CDC-ACM) | Виртуальный COM | Многие современные |

Физический интерфейс меняется, **команды остаются теми же**.

## Разница между АКИП и Gratten

Это ключевой момент твоего проекта:

### Gratten GA1483 — стандартный SCPI

```
:FREQuency:CW 1000000000Hz   ← стандарт IEEE 1288
:POWer:LEVEL -10DBM
:OUTPut:STATE ON
```

Практически идентичен RIGOL DSG800 и Siglent SSG3000X. Документации много.

### АКИП-3417 (Suin TFG) — проприетарный SCPI

```
FREQ:CHA 1000                ← нет ведущего ":"
VOLT:CHA 1.0 VPP             ← CHA/CHB вместо :CHANnel1
OUTP:CHB ON
```

Suin придумал свой синтаксис каналов. Документации мало. Именно поэтому ресерч агента искал OEM-производителя (нашёл: Suin Instruments).

## Как устроен твой код

```
IAkipController
  ├── setFrequency(channel, hz)  ← ты вызываешь это
  │
  ├── AkipFacade
  │   └── m_usb.sendScpiCommand("FREQ:CHA 1000")   ← это уходит в USB
  │
  └── GrattenGa1483Controller
      └── m_lan.sendScpiCommand(":FREQuency:CW 1000Hz")  ← это уходит в TCP
```

Благодаря полиморфизму, MainWindow не знает какую из двух команд отправить — это забота контроллера.

---

## Практика: querySystemError

В Gratten есть команда `:SYSTem:ERRor?`. Если всё хорошо — отвечает `+0,"No error"`. Если что-то пошло не так — возвращает код ошибки.

В `AbstractScpiController` это используется в `verifySetCommand()` — после отправки команды можно спросить "а вдруг прибор не принял?".
